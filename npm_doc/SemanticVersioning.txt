**Semantic Versioning (SemVer)** là chuẩn để đánh số phiên bản phần mềm, đặc biệt phổ biến trong hệ sinh thái npm, giúp người dùng và nhà phát triển hiểu **mức độ thay đổi và tương thích** của một package.

---

### 1. Cấu trúc phiên bản

Phiên bản Semantic Versioning có dạng:

```
MAJOR.MINOR.PATCH
```

* **MAJOR**: Thay đổi không tương thích ngược (breaking changes).
* **MINOR**: Thêm tính năng mới mà vẫn tương thích ngược (backwards-compatible).
* **PATCH**: Sửa lỗi, không ảnh hưởng API.

Ví dụ:

```
1.2.3
```

* `1` → major version
* `2` → minor version
* `3` → patch version

---

### 2. Nguyên tắc cơ bản

1. **Tăng MAJOR** khi thay đổi API làm hỏng khả năng tương thích cũ.
2. **Tăng MINOR** khi thêm tính năng mới mà code cũ vẫn chạy được.
3. **Tăng PATCH** khi chỉ sửa lỗi nhỏ, không ảnh hưởng tới API.

---

### 3. Lợi ích

* Giúp người dùng npm **biết trước mức độ rủi ro khi cập nhật** package.
* Dễ dàng quản lý dependency, ví dụ: `"lodash": "^4.17.21"` → tự động cập nhật patch/minor nhưng không nâng major.
